aikido init

Initialize project. Create all dirs, default templates, contracts, git init.

$ aikido init my-company
$ aikido init my-company --no-git
$ aikido init my-company --remote git@github.com:me/my-company.git
Argument / OptionDefaultDescription
name (required) — Project directory name (required)
--template default Starter template set
--git / --no-git --git Initialize a git repo + first commit
--remote — Add an origin remote URL

aikido weekly

Aggregate daily logs into weekly summaries, then transpile agents.

$ aikido weekly
$ aikido weekly --week 2026-W29
$ aikido weekly --no-build
$ aikido weekly --partial
Argument / OptionDefaultDescription
--build / --no-build --build Transpile agents after aggregating
--partial / --no-partial false Treat build failures as skips, not errors
--week current ISO week Week id, e.g. 2026-W29

aikido build_agent

Transpile Jinja2 template + contract -> flat Markdown artifact in 06_dist/.

$ aikido build_agent founder_weekly_review --week 2026-W29
$ aikido build_agent founder_weekly_review --week 2026-W29 --focus-area gtm
$ aikido build_agent founder_weekly_review --emergency
Argument / OptionDefaultDescription
agent (required) — Agent name, matching 05_agents/{agent}.prompt.md (required)
--week — Week of context to inline
--focus-area — product or gtm — pulls in extra domain context
--emergency / --no-emergency false Flag the build as an emergency variant
Exits non-zero and prints the failing stage (pre_transpile, transpile, or post_transpile) when a build fails.
