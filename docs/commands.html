Command reference

All 12 commands. Options use Typer conventions — underscores become hyphens (--focus-area), and booleans get a --no- negation (--no-git).

CommandWhat it does
initScaffold dirs, default templates/contracts/control-plane, git init
dailyInteractive daily capture → weekly cadence
weeklyAggregate daily logs, then transpile agents
build_agentTranspile one template + contract → flat Markdown
updateUpdate domain knowledge or support themes
syncPull from Notion/Linear/Slack/Stripe
archiveCompress old weekly files
onboardGenerate an onboarding brief
investorGenerate an investor update
dashboardKnowledge-system health dashboard
validateCheck includes, variables, contracts, stale files
configManage .aikido/config.yaml

aikido init

Initialize a project: create every directory, write the default templates, contracts, and control-plane files, and initialize git.

$ aikido init my-company
$ aikido init my-company --no-git
$ aikido init my-company --remote git@github.com:me/my-company.git
OptionDefaultDescription
name—Project directory name (required)
--templatedefaultStarter template set
--git / --no-git--gitInitialize a git repo + first commit
--remote—Add an origin remote URL

aikido daily

Guided daily capture. Prompts for day type, ships, blockers, customer moment, takeaway, and people; auto-tags from content; saves to 01_weekly_cadence/{week}/{date}_daily.md.

$ aikido daily
$ aikido daily --retroactive 2026-07-14
OptionDefaultDescription
--guided / --no-guided--guidedInteractive prompt flow
--retroactivetodayCapture for a past date (YYYY-MM-DD)

aikido weekly

Aggregate the week's daily logs into decisions.md, blockers.md, customer_signals.md, and daily_summary.md, then transpile every agent that has a matching contract.

$ aikido weekly
$ aikido weekly --week 2026-W29
$ aikido weekly --no-build          # aggregate only
$ aikido weekly --partial            # skip agents that fail to build
OptionDefaultDescription
--weekcurrent ISO weekWeek id, e.g. 2026-W29
--build / --no-build--buildTranspile agents after aggregating
--partialfalseTreat build failures as skips, not errors

aikido build_agent

Transpile a single Jinja2 template + JSON contract into a flat Markdown artifact in 06_dist/.

$ aikido build_agent founder_weekly_review --week 2026-W29
$ aikido build_agent founder_weekly_review --week 2026-W29 --focus-area gtm
$ aikido build_agent founder_weekly_review --emergency
OptionDefaultDescription
agent—Agent name, matching 05_agents/{agent}.prompt.md (required)
--week—Week of context to inline
--focus-area—product or gtm — pulls in extra domain context
--emergencyfalseFlag the build as an emergency variant
Exits non-zero and prints the failing stage (pre_transpile, transpile, or post_transpile) when a build fails.

aikido update

Append an update to a domain-knowledge or support-theme file, stamped with its source.

$ aikido update 02_domain_knowledge/product/roadmap.md --from-source linear
$ aikido update 03_support_tickets/themes/churn.md --tickets "T-101,T-102"
OptionDescription
pathFile to update (required)
--from-sourceNamed source (e.g. linear, notion)
--urlSource URL
--ticketsComma-separated ticket ids

aikido sync

Pull from an external source into the knowledge base. Routes to a sensible default file per source, or override with --to-path.

$ aikido sync notion --since 2026-07-01
$ aikido sync slack --channel "#feedback"
$ aikido sync stripe --to-path 02_domain_knowledge/go_to_market/pricing.md
OptionDescription
from_sourcenotion · linear · slack · stripe (required)
--to-pathOverride the destination file
--channelSlack channel to pull
--sinceOnly pull items since this date

aikido archive

Move old weekly directories into 07_archive/. Accepts a single week, a comma list, or a range (2026-W01..2026-W12).

$ aikido archive 2026-W29
$ aikido archive 2026-W01..2026-W12
$ aikido archive 2026-W29 --no-compress   # write a summary instead of a tarball
OptionDefaultDescription
weeks—Week id, list, or range (required)
--compress / --no-compress--compressTar.gz vs. Markdown summary

aikido onboard

Generate an onboarding brief for a new team member, seeded from the control plane and available weekly context.

$ aikido onboard "Jane Doe" engineer
$ aikido onboard "Sam Lee" "gtm lead" --from-week 2026-W20
OptionDescription
name / roleNew hire's name and role (required)
--from-weekInclude context from this week onward

aikido investor

Generate an investor update from captured customer signals.

$ aikido investor
$ aikido investor --since last_board_meeting --format markdown
OptionDefaultDescription
--brief / --no-brief--briefConcise vs. full update
--sincelast_board_meetingReporting window
--formatemailemail or markdown

aikido dashboard

A visual overview of knowledge-system health: weeks captured, agents defined, artifacts built, latest week, and optionally stale files.

$ aikido dashboard
$ aikido dashboard --my-agents --stale
OptionDefaultDescription
--my-agentsfalseList defined agents
--status / --no-status--statusShow the status table
--stalefalseCount stale files

aikido validate

Check the knowledge base for broken includes, undefined variables, contract mismatches, and stale files. Exits non-zero if any non-stale issue is found.

$ aikido validate
$ aikido validate --path 05_agents/founder_weekly_review.prompt.md
OptionDescription
--pathLimit validation to one template
--fixReserved for auto-fixes

aikido config

Read and write .aikido/config.yaml using dotted keys.

$ aikido config --list-all
$ aikido config defaults.agent
$ aikido config defaults.focus_area product
ArgumentDescription
keyDotted key to read (e.g. git.remote)
valueProvide with a key to set it
--list-allPrint the whole config as a table
